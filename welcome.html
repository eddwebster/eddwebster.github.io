Hi, I'm Edd.

As a Data Scientist with five years of professional experience, my job is to help organisations reduce their time to insight through a data-driven approach.

After graduating in 2016 from Queen Mary University of London with an MSci degree in Chemistry, I have worked as a Data Scientist for a number of best-in-class organisations including the LEGO Group, Analytics FC, West Ham United Football Club, and City Football Group. I now work as the Lead Data Scientist for the First Team at Leicester City Football Club, helping to drive the delivery of data-driven insights for performance analysis, player recruitment, sports science, and medical aspects of the club's operation.

My professional interests include football analytics, applied data science, machine learning, and data visualisation. In my free time, I maintain a football_analytics GitHub repository of open source analysis, that includes projects for model building including xG models; working with tracking data including the application of pitch control models; the application of action value frameworks including xT, and VAEP; and creating football intelligence reporting tools using Tableau. This repository also includes a curated list of publicly available resources such as blog posts, libaries, and the latest academic literature, to help people get started with football analytics.

Full details of my previous experience and publicly available analysis can be found on my profiles for LinkedIn, GitHub, Tableau Public, as well as my CV, Football Data Science portfolio, and the following article by Training Ground Guru that discusses my thoughts and experience regarding football analytics and data science.

Data-Driven Recruitment of Ball-Playing Center Backs

Through data engineering of StatsBomb Event data and the creation of Football Intelligence reporting tools in Tableau, this project is a data-driven analysis of the best ball-playing center backs from the 2018 FIFA World Cup.

Description

A recruitment department of a club are looking to sign a ball-playing center back that fits within their progressive playing style. As part of their recruitment strategy, the department require data-driven identification of the top three ball-playing center backs from the 2018 FIFA World Cup.

Of these three recommendations:

  • At least one should be a player aged 23 or younger (at the time of the tournament).
  • Due to budgetary restrictions, one player should come from a domestic team playing outside the 'Big 5' European leagues or with a estimated value of less than £5mil as per TransferMarkt.

This project engineers and aggregates World Cup Event data provided by StatsBomb and enriches it with scraped player bio and valuation data from TransferMarkt, to identify the high-performing ball-playing center backs through the recommendation and analysis of suitable KPIs.

From the resulting dataset, professional Football Intelliigence tools are created in Tableau, for use in a professional football club environment, to enable the democratisation of data for football practitioners, helping to reduce their time to insight.

The Tableau dashboards can be found on Tableau Public and the Python source code to engineer the data can be found as a Jupyter notebook in my football_analytics GitHub repository. A full slide deck of this analysis along with my other football analytics projects is available in Google Slides.

Pitch Control Models from Tracking Data

Application of football Tracking data provided by Signality, to create Pitch Control models, which are in turn combined with Expected Possession Value (EPV) frameworks, to determine the Expected Value-Added. This data is then later synchronised with video footage to bring together human expertise and data-driven insight.

Description

This project takes the publicly available Tracking data for three Hammarby matches in the Swedish Allsvenskan, provided by Signality, to analyse moments in play, create Pitch Control models, and determine the Expected Value-Added by combining Pitch Control models and Expected Possession Value (EPV) frameworks.

This project is conducted entirely using Python in a Jupyter notebook with pandas and matplotlib libraries.

The source code for this project can be found in my GitHub repository. A full slide deck of football data science projects, including working with Tracking data, is available in Google Slides. If you would like to work together on a similar project, email me at edd.j.webster@gmail.com.

Training Expected Goals Models from Event Data

Deriving Expected Goals (xG) models from Event data using Logistic Regression and Gradient Boosted Algorithms, to determine the probability of a goal-scoring chance resulting in a goal.

Description

This project takes the publicly available Event data from StatsBomb and Wyscout, to train Expected Goals (xG) models using Logistic Regression and Gradient Boosted Algorithms. From these models, we can then determine the probability of a goal-scoring chance resulting in a goal.

This project is conducted entirely using Python in Jupyter notebooks with pandas for data engineering, matplotlib for visualisation, and scikit-learn and XGBoost for the application of machine learning.

The source code for this project can be found in my GitHub repository. A full slide deck of football data science projects, including training Expected Goals models, is available in Google Slides. If you would like to work together on a similar project, email me at edd.j.webster@gmail.com.
